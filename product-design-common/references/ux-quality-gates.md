# UX Quality Gates

Review concepts against:
- audience/problem fit
- value proposition clarity
- information hierarchy
- primary action clarity
- cognitive load
- trust and credibility
- accessibility basics
- responsive feasibility
- implementation feasibility
- evidence strength

Severity:
- blocker: concept likely misleads, fails core task, or cannot be trusted
- major: hurts comprehension, trust, conversion, or feasibility
- minor: polish or local clarity issue

Verdicts:
- accept: concept is good enough for handoff
- revise concept: source/flow/spec needs changes before another visual pass
- regenerate concept: Stitch/visual output misses the intended concept
- hand off: sufficient for downstream design-skills/frontend/stakeholder review
- blocked: missing evidence/input or unsafe external action

Example finding:
`major — value prop is generic; user cannot tell why this beats existing dashboard. Revise brief and hero hierarchy before Stitch.`
