# Product Concept Brief

Use to create the upstream brief for product-design concept work. This is not a full PRD.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/market-context-rules.md`
- `../product-design-common/references/output-shapes.md`
- `../product-design-common/references/safety-boundaries.md`

Workflow:
1. Read available source/context notes.
2. Define target user, problem/job, value proposition, constraints, and non-goals.
3. Mark confirmed facts vs assumptions.
4. Define success criteria and the smallest useful concept scope.
5. Hand off to opportunity or journey/spec lanes.

Output:
- concept brief
- user / problem / job
- value proposition
- constraints and non-goals
- success criteria
- assumptions and evidence gaps
- risks
