# Handoff Packet

A product-design handoff should be small and useful.

## Canonical shape

```markdown
# Product-design handoff: <concept>

## Source-truth summary
- User / problem / job:
- Value proposition:
- Selected opportunity:
- Journey / flow:
- Page specs:

## Concept evidence
- Stitch concept artifacts:
- Generated prompt / degraded fallback:
- Review verdict:

## Risks and open questions
- UX risks:
- Positioning/marketing risks:
- Assumptions:
- Missing evidence:

## Recommended downstream lane
- design-skills | coding | marketing/GTM | research | stop
```

Mark generated Stitch screens as concept evidence, not source truth.
