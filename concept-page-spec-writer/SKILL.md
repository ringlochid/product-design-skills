# Concept Page Spec Writer

Use to define what a concept screen must communicate and support before Stitch concept generation.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/concept-page-spec-rules.md`
- `../product-design-common/references/journey-flow-rules.md`
- `../product-design-common/references/output-shapes.md`
- `../product-design-common/references/industry-quality-bar.md`
- `../product-design-common/references/safety-boundaries.md`

Workflow:
1. Read the concept brief and journey/flow notes.
2. Define page goal, audience moment, hierarchy, sections, actions, and data needs.
3. Tie key screen decisions to evidence, standards, or explicit assumptions.
4. Include key states, accessibility basics, and responsive intent at concept fidelity.
5. Keep copy as intent-level unless exact copy is required.
6. Hand off to Stitch concept generation or review.

Output:
- page/screen spec
- hierarchy and sections
- primary actions and data
- states / edge cases
- responsive intent
- trust/privacy/data constraints when relevant
- standards/evidence used
- assumptions and missing evidence
- risks
