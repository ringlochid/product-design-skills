# Concept Review Gate

Use to critique product-design concepts, sketches, or Stitch prototype screens before handoff or deeper design work.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/ux-quality-gates.md`
- `../product-design-common/references/positioning-marketing-risk.md`
- `../product-design-common/references/output-shapes.md`
- `../product-design-common/references/safety-boundaries.md`

Workflow:
1. Identify target user, product promise, and success criterion.
2. Review clarity, hierarchy, task flow, trust, accessibility basics, and feasibility.
3. Check positioning/marketing risk without turning into a campaign plan.
4. Classify issues by severity and next action.
5. Return accept, revise concept, regenerate concept, hand off, or blocked.

Output:
- artifact/concept reviewed
- evidence used
- UX findings and severity
- positioning/marketing risks
- verdict and next action
- missing evidence
- assumptions
