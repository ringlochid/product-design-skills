# Stitch Concept Generator

Use Stitch only to illustrate product ideas and quick concept prototypes. This is not the full design-skills E2E conductor.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/stitch-concept-rules.md`
- `../product-design-common/references/concept-page-spec-rules.md`
- `../product-design-common/references/safety-boundaries.md`
- `../product-design-common/references/output-shapes.md`

Workflow:
1. Confirm concept brief/page spec is good enough.
2. Build a concise Stitch prompt from product intent, hierarchy, and style direction.
3. Ask before external Stitch create/generate/edit actions when required by environment policy.
4. Treat generated screens as concept evidence, not source truth.
5. Return artifacts to concept review.

Output:
- concept prompt / generation brief
- Stitch action requested or performed
- concept artifact links/paths if any
- caveats and degraded evidence
- review handoff
- risks / blocked conditions
- assumptions
