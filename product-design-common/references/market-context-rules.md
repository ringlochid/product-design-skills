# Market Context Rules

Use market context to sharpen product/design choices, not to run a news desk.

Capture:
- target segment and problem intensity
- competitor/reference patterns
- pricing/packaging constraints when relevant
- customer quotes or support signals when supplied
- regulatory/trust/compliance constraints when visible

Separate:
- fact
- source claim
- community/social signal
- inference

Stop when the context can support a brief or when missing evidence would change the concept materially.
