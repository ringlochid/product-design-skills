# Opportunity Framing

Use when an idea, feature request, or stakeholder ask needs problem framing before concept screens.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/opportunity-framing-rules.md`
- `../product-design-common/references/market-context-rules.md`
- `../product-design-common/references/output-shapes.md`
- `../product-design-common/references/safety-boundaries.md`

Workflow:
1. Name the desired outcome and metric if known.
2. Generate 2-4 customer opportunities/problems, not feature lists.
3. Map solution hypotheses only after opportunities are clear.
4. Pick the best first concept/test by evidence, impact, and feasibility.
5. Stop before page specification unless the concept target is clear.

Output:
- desired outcome
- opportunities / pains
- solution hypotheses
- validation test ideas
- recommendation and tradeoffs
- assumptions and risks
- evidence used
