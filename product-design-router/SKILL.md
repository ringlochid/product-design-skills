# Product Design Router

Use as the thin front door for product-design concept/prototype work. Return a lane plan, not the whole artifact.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/routing-matrix.md`
- `../product-design-common/references/safety-boundaries.md`

Workflow:
1. State the product idea/request, audience, stage, and output need.
2. Pick effort: quick, standard, deep, or concept-prototype.
3. Choose one first lane unless the request truly spans stages.
4. Name optional downstream lanes only when they add evidence or a visual concept.
5. Stop at the handoff boundary; do not route into the full design E2E conductor unless explicitly asked.
6. For explicit end-to-end concept-prototype requests, plan the full chain: context → brief → opportunity → journey → spec → Stitch concept → review → handoff.

Output:
- selected lane(s)
- effort tier
- product-design plan
- assumptions and missing input
- evidence needed
- risks / blocked conditions
