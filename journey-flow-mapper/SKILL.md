# Journey Flow Mapper

Use to translate a product concept into user journeys and flow structure before page/screen specs.

Read first:
- `../product-design-common/references/product-design-contract.md`
- `../product-design-common/references/journey-flow-rules.md`
- `../product-design-common/references/output-shapes.md`
- `../product-design-common/references/safety-boundaries.md`

Workflow:
1. Pick one primary persona/use case.
2. Map start, trigger, steps, decisions, success, and failure paths.
3. Include loading, empty, error, permission, and edge states when relevant.
4. Mark friction, trust, and conversion risks.
5. Hand off to concept page spec.

Output:
- journey scope
- primary flow map
- decision points and states
- friction / trust risks
- evidence and assumptions
- next page/spec lane
