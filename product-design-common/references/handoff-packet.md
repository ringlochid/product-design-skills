# Handoff Packet

A product-design handoff is a compact transfer package, not a second report.

Include:
- artifact inventory and status
- source-truth summary
- beachhead segment, launch region, use moment, excluded users
- strongest direct opponent, current workaround, opponent gap, novelty delta hypothesis
- key decisions with evidence/standard basis
- concept image paths, critique verdict, and story/screen ID mapping
- UI concept artifact paths/screenshots, or degraded/blocked reason
- promotion verdict and source-link status
- risks, open questions, assumptions, and missing evidence
- recommended downstream lane
- next validation step

Separate source truth from generated artifacts. Generated images and UI concepts are communication evidence, not user validation.

Promotion handoff requires `14-source-link-check.txt` and `15-promotion-verdict.md` with `promotion_eligible: true` only after source links, concept images, UI concept screenshots, visual/a11y review, and independent audit pass.

Do not hand off as implementation-ready unless the UI concept gate and artifact checker pass.
