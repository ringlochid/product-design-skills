# Karpathy-Style Product Design Rules

Use agent discipline, not ceremony.

Rules:
- evidence before confidence
- problem before screens
- one real beachhead before broad audiences
- one sharp user path before broad IA
- simple artifact before large design system
- UI concept proof before polished prose
- name non-goals and delete ornamental scope
- use the smallest relevant standard/source subset
- stop when evidence, authority, or artifact access is missing
- verify before saying done

Execution style:
- leaf asks stay leaf-sized; do not inflate them into full runs
- full promotion packages use the workflow conductor and no extra files beyond the contract
- degraded means degraded; do not hide it behind positive wording
- do not mutate external tools without approval
- generated artifacts need review before promotion
