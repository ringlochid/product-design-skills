# Product Design Contract

Product-design turns market/customer/product intent into concept-ready UX source truth and lightweight prototypes.

## Lifecycle

1. Clarify audience, problem, outcome, and use case.
2. Read only enough market/context evidence to avoid fantasy design.
3. Frame opportunity before solution screens.
4. Map the core journey/flow and important states.
5. Write lightweight concept page specs.
6. Use Stitch only for concept illustration when useful.
7. Review UX, positioning, risk, and evidence.
8. Hand off cleanly to design-skills, frontend, or stakeholders.

## Boundaries

- Product-design may create concept prompts/screens.
- Full design-skills owns serious design repo lifecycle, responsive repair, generation packs, production handoff, and Stitch artifact governance.
- Coding skills own implementation.
- Future marketing/GTM skills own campaigns, channels, launch calendars, and ad systems.

Stop when source evidence is too thin, the next step requires external writes/credentials, or the user needs full E2E design execution.
